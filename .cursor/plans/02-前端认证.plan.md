# Dream Log 统一认证系统实现计划

## 架构设计

```mermaid
flowchart TD
    Start[用户访问 /auth] --> InputEmail[输入邮箱]
    InputEmail --> EmailValidation{前端邮箱格式验证}
    EmailValidation -->|无效| ShowError[显示错误提示]
    EmailValidation -->|有效| CheckEmail[向后端查询邮箱状态]
    
    CheckEmail --> IsExist{邮箱是否已注册}
    
    IsExist -->|是-已注册| LoginFlow[登录流程]
    IsExist -->|否-未注册| SignupFlow[注册流程]
    
    LoginFlow --> LoginMethod{选择登录方式}
    LoginMethod -->|密码登录| InputPassword[输入密码]
    LoginMethod -->|邮箱验证码| SendLoginCode[发送验证码]
    
    InputPassword --> ValidatePassword{验证密码}
    ValidatePassword -->|成功| LoginSuccess[登录成功]
    ValidatePassword -->|失败| PasswordError[密码错误提示]
    PasswordError --> ForgotPassword[忘记密码?]
    ForgotPassword --> SendLoginCode
    
    SendLoginCode --> InputLoginCode[输入验证码]
    InputLoginCode --> VerifyLoginCode{验证验证码}
    VerifyLoginCode -->|正确| LoginSuccess
    VerifyLoginCode -->|错误| LoginCodeError[显示错误,允许重发]
    
    SignupFlow --> SignupMethod{选择注册方式}
    SignupMethod -->|邮箱验证码| SendSignupCode[发送验证码]
    SignupMethod -->|创建密码| InputNewPassword[输入密码]
    
    InputNewPassword --> ValidatePasswordStrength{密码强度验证}
    ValidatePasswordStrength -->|不合规| PasswordStrengthError[显示密码要求]
    ValidatePasswordStrength -->|合规| SendSignupCodeAfterPassword[发送验证码]
    SendSignupCodeAfterPassword --> InputSignupCodeWithPassword[输入验证码]
    InputSignupCodeWithPassword --> VerifyAndRegister{验证验证码}
    
    SendSignupCode --> InputSignupCode[输入验证码]
    InputSignupCode --> VerifySignupCode{验证验证码}
    VerifySignupCode -->|正确| SignupSuccess[注册成功并自动登录]
    VerifySignupCode -->|错误| SignupCodeError[显示错误,允许重发]
    
    VerifyAndRegister -->|正确| SignupSuccess
    VerifyAndRegister -->|错误| SignupCodeError
    
    LoginSuccess --> Dashboard[跳转到应用首页]
    SignupSuccess --> Dashboard
```

## 实现补充说明

### 安全性考虑

1. **验证码保护**: 前端限制重发间隔(60 秒),后端应有频率限制
2. **密码要求**: 8 位以上,需包含大写、小写、数字、特殊符号中至少 3 类
3. **Token 存储**: 使用 localStorage 存储 JWT token
4. **HTTPS**: 生产环境必须使用 HTTPS

### 流程细节补充

1. **邮箱验证码注册**: 用户输入邮箱 → 发送验证码 → 输入验证码 → 注册成功自动登录 → 可在用户设置中添加密码
2. **密码注册**: 用户输入邮箱 → 创建密码(需验证强度) → 发送验证码 → 输入验证码 → 注册成功
3. **密码登录**: 输入邮箱 → 输入密码 → 登录成功
4. **邮箱验证码登录**: 输入邮箱 → 选择验证码方式 → 发送验证码 → 输入验证码 → 登录成功
5. **忘记密码**: 在密码登录页点击 "忘记密码" → 切换到验证码流程 → 验证成功后可选择是否设置新密码

### Google OAuth 流程

1. 前端点击 Google 按钮 → 调用 `initiateGoogleOAuth` 获取授权 URL
2. 重定向到 Google 授权页面
3. 用户授权后重定向到 `/auth/callback/google?code=xxx`
4. 回调页面调用后端接口用 code 换取 token
5. 保存 token 并跳转到应用首页
