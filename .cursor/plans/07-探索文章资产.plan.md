# 三模块高质量文章内容资产构建计划

## 核心原则

### 文章结构必须一致吗？

**不必强制统一，但需要有「一致的设计意图」。**

- 强制相同结构 → 模板感强、阅读体验机械
- 完全无结构 → 用户不知道期待什么，产品缺乏可信感

**推荐：模块内部一致，模块之间服务于不同目的，结构自然不同**

- 梦境科学基础 → 每篇：「是什么 + 科学解释 + 与你的关联」
- 噩梦应对指南 → 每篇：「问题描述 + 原因 + 可操作建议」
- 梦境改善指南 → 每篇：「目标 + 方法步骤 + 预期效果」

当前数据库 `content` 字段为 JSONB，已支持灵活结构，**无需改表**。

### 应该收集哪些领域的权威来源？

跨领域收集，不仅限于梦境本身：

- 睡眠科学
- 神经科学
- 情绪心理学
- 临床研究

## 完整工作流（优化后）

```
Step 1：Perplexity Deep Research × 3次（每模块一次提问，覆盖6个主题）
         ↓ 保存到 backend/src/knowledge_sources/science.md / nightmare.md / improvement.md

Step 2：AI 批量整合 × 3次（每模块一次，输出完整 JSON 数组）
         ↓ 合并3个 JSON 数组 → scripts/articles/articles_input.json（18篇）

Step 3：python scripts/articles/import_articles.py（一键入库）
         ↓ 读取 articles_input.json → upsert → exploration_articles 表

Step 4：前端验证
         ↓ 访问 /exploration/science、/nightmare、/improvement 确认显示正确
```

权威性来自 Perplexity 的来源聚合，GPT 负责整合和语气转换，不创造新知识。如内容有误，更新 JSON 后重新运行脚本 upsert 即可。

## AI 整合输出格式：直接 JSON

**不使用纯文本或 Markdown**，AI 直接输出符合数据库结构的 JSON，原因：

- 无需中间格式转换
- 可直接合并追加到 `articles_input.json`
- 脚本直接读取，零额外处理

**JSON 格式规范**（前端 `article-page.tsx` 消费 `section` 和 `content.body`）：

```json
[
  {
    "module": "science",
    "section": "我们为什么做梦？",
    "order_index": 1,
    "content": {
      "title": "我们为什么做梦？",
      "body": "正文第一段。\n\n正文第二段。\n\n正文第三段。",
      "expandable": true,
      "source_note": "基于睡眠科学研究整合"
    }
  }
]
```

`body` 字段使用**纯文本 + `\n\n` 段落分隔**，前端已有 `whitespace-pre-line` 样式直接渲染，无需 Markdown 或 HTML。

## 数据库导入脚本

`backend/scripts/articles/import_articles.py`，参考 `import_symbols.py` 模式：

- 读取 `scripts/articles/articles_input.json`
- 按 `(module, section)` 检查是否已存在（upsert 逻辑）
- 一次性 `commit`，无需人工确认，重复运行安全
- 输出新增/更新/失败统计

**运行方式**（在 backend 根目录）：

```bash
python scripts/articles/import_articles.py
```
