根据 `frontend/src/contexts/` 目录中的文件，我来为您介绍这些上下文（Context）的用途：

## 1. `AuthContext.jsx` - 认证上下文
**用途**：管理用户认证状态和认证相关操作
**主要功能**：
- **状态管理**：用户信息、加载状态、认证状态、访问令牌
- **认证操作**：统一登录、注册、密码重置、登出
- **令牌管理**：自动刷新、有效期检查、清除过期令牌
- **事件监听**：监听认证相关事件（登出、用户更新等）
- **初始化**：应用启动时自动检查认证状态

**适用场景**：整个应用的认证状态管理，任何需要知道用户是否登录的组件

## 2. `FeatureFlagContext.jsx` - 功能开关上下文
**用途**：动态控制应用功能的启用/禁用状态
**主要功能**：
- **功能开关**：从后端API获取功能开关配置
- **动态控制**：SMS服务、邮件服务、密码登录等功能的开关
- **方法获取**：获取可用的登录方式、注册方式、密码重置方式
- **默认配置**：API失败时使用默认配置
- **React 19支持**：使用新的 `use` API 处理异步数据

**适用场景**：根据后端配置动态显示/隐藏功能，A/B测试，灰度发布

## 3. `I18nContext.jsx` - 国际化上下文
**用途**：提供多语言支持和本地化功能
**主要功能**：
- **翻译功能**：支持多语言文本翻译
- **语言切换**：动态切换应用语言
- **格式化工具**：日期、时间、数字的本地化格式化
- **命名空间**：支持多个翻译文件命名空间
- **向后兼容**：兼容现有的翻译键格式
- **持久化**：语言选择保存到本地存储

**适用场景**：多语言应用，国际化支持，本地化显示

## 4. `ThemeContext.jsx` - 主题上下文
**用途**：管理应用的亮色/暗色主题切换
**主要功能**：
- **主题切换**：在亮色和暗色主题之间切换
- **系统检测**：自动检测系统主题偏好
- **持久化**：主题选择保存到本地存储
- **动态应用**：实时应用主题到DOM
- **响应式**：监听系统主题变化并自动调整

**适用场景**：支持明暗主题的应用，提升用户体验，减少眼部疲劳

## 上下文的作用和优势：

### 1. **状态共享**
- 避免 prop drilling（属性传递地狱）
- 在组件树中任意位置访问共享状态
- 统一管理全局状态

### 2. **关注点分离**
- 每个上下文专注于特定功能领域
- 代码组织更清晰
- 便于维护和测试

### 3. **性能优化**
- 使用 `useMemo` 和 `useCallback` 优化渲染
- 避免不必要的重新渲染
- 按需更新相关组件

### 4. **类型安全**
- 提供统一的接口和类型定义
- 减少运行时错误
- 更好的开发体验

### 5. **可扩展性**
- 易于添加新功能
- 支持插件化架构
- 便于团队协作

这些上下文共同构成了应用的核心状态管理层，为整个应用提供了认证、功能控制、国际化和主题等基础服务，是现代 React 应用架构的重要组成部分。
