根据 `frontend/src/hooks/` 目录中的文件，我来为您介绍这些自定义钩子的用途：

## 1. `useAutoScroll.js` - 自动滚动钩子
**用途**：提供自动滚动到底部功能，当用户手动滚动时会禁用自动滚动
**主要功能**：
- 自动检测内容变化并滚动到底部
- 监听用户手动滚动行为，避免干扰
- 支持平滑滚动和即时滚动
- 监听容器大小变化
- 提供手动控制滚动的方法

**适用场景**：聊天界面、实时消息显示、日志查看等需要自动滚动的地方

## 2. `useDebounce.js` - 防抖钩子
**用途**：对频繁变化的值进行防抖处理，避免过度触发副作用
**主要功能**：
- 延迟更新值，直到指定时间内没有新的变化
- 可配置延迟时间
- 自动清理定时器

**适用场景**：搜索输入框、API 请求优化、表单验证等

## 3. `useLocalStorage.js` - 本地存储钩子
**用途**：提供与 localStorage 交互的便捷接口
**主要功能**：
- 自动序列化/反序列化 JSON 数据
- 错误处理，避免 localStorage 异常导致应用崩溃
- 支持函数式更新（类似 useState）
- 支持删除存储项

**适用场景**：用户设置保存、表单数据持久化、缓存数据等

## 4. `useProgressiveLoading.js` - 渐进式加载钩子
**用途**：实现大数据集的渐进式加载，提升用户体验
**主要功能**：
- 分批加载数据，避免一次性渲染大量内容
- 自动检测数据变化并重置状态
- 小数据集直接显示，大数据集启用渐进加载
- 提供加载进度和状态信息

**适用场景**：长列表渲染、大量数据展示、性能优化等

## 5. `useUndoRedo.js` - 撤销/重做钩子
**用途**：提供撤销和重做功能，支持操作历史管理
**主要功能**：
- 维护操作历史记录
- 支持撤销/重做操作
- 限制历史记录大小
- 提供操作描述和状态信息
- 专门处理图片删除的撤销/重做

**适用场景**：编辑器、表单操作、图片管理等需要撤销功能的地方

## 6. `useWebSocket.js` - WebSocket 连接钩子
**用途**：管理 WebSocket 连接，提供实时通信功能
**主要功能**：
- 自动连接和重连机制
- 身份验证（JWT Token）
- 心跳检测保持连接活跃
- 连接状态管理
- 消息发送和接收

**适用场景**：实时聊天、实时通知、实时数据更新等

## 7. `index.js` - 钩子统一导出
**用途**：集中管理所有自定义钩子的导出
**主要功能**：
- 统一导出所有钩子
- 提供默认导出对象
- 便于其他模块导入使用

这些钩子都是为了提高代码复用性、简化组件逻辑、提供通用功能而设计的，遵循 React Hooks 的最佳实践，让您的应用更加模块化和可维护。
