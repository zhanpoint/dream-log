根据 `frontend/src/constants/` 目录中的文件，我来为您介绍这些常量文件的用途：

## 1. `api.js` - API 端点常量
**用途**：集中管理所有 API 路径，确保与后端 RESTful 规范一致
**主要功能**：
- **认证相关端点**：登录、令牌刷新、令牌验证
- **用户管理端点**：用户详情、用户列表、邮箱管理、密码管理
- **验证码端点**：短信验证码、邮箱验证码
- **系统端点**：功能开关配置
- **梦境相关端点**：梦境列表、详情、分类、标签
- **文件上传端点**：上传签名、完成上传、标记删除

**优势**：统一管理 API 路径，便于维护和修改，避免硬编码

## 2. `dream.js` - 梦境相关常量
**用途**：管理梦境相关的配置和选项
**主要功能**：
- **梦境分类**：16种梦境类型（普通梦境、清醒梦、噩梦等）
- **情绪配置**：5种情绪等级（非常消极到非常积极）
- **睡眠质量**：5个等级（很差到很好）
- **隐私设置**：私人、公开、好友可见三种隐私级别
- **标签类型**：6种标签分类（情感、角色、地点等）
- **国际化支持**：提供多语言配置函数

**优势**：统一管理梦境相关的业务逻辑，支持多语言，便于扩展

## 3. `routes.js` - 路由路径常量
**用途**：集中管理所有页面路由和导航配置
**主要功能**：
- **页面路由**：定义所有页面的路径常量
- **路由生成器**：动态生成带参数的路由
- **导航菜单**：配置主导航菜单项
- **权限控制**：区分公开路由和需要认证的路由
- **面包屑配置**：定义页面导航路径
- **页面标题**：SEO 友好的页面标题配置

**优势**：统一管理路由，便于维护和 SEO 优化

## 4. `ui.js` - UI 相关常量
**用途**：管理界面元素的配置项和规则
**主要功能**：
- **表单验证规则**：用户名、密码、手机号、邮箱等验证规则
- **分页配置**：默认页面大小、可选页面大小、最大可见页数
- **文件上传配置**：文件大小限制、允许的文件类型
- **通知配置**：不同类型通知的显示时长

**优势**：统一管理 UI 规则，确保用户体验一致性

## 5. `index.js` - 常量统一导出
**用途**：作为常量的统一入口，集中导出所有常量
**主要功能**：
- **统一导入**：从各个常量文件导入所有常量
- **统一导出**：提供命名导出和默认导出
- **便捷访问**：通过一个文件访问所有常量

**优势**：简化导入语句，提供统一的常量访问接口

## 常量的作用和优势：

### 1. **代码维护性**
- 集中管理配置，修改时只需改一处
- 避免魔法数字和硬编码字符串
- 提高代码可读性

### 2. **类型安全**
- 提供统一的常量定义
- 减少拼写错误
- 便于 IDE 自动补全

### 3. **业务逻辑分离**
- 将配置与业务逻辑分离
- 便于非开发人员修改配置
- 支持 A/B 测试和功能开关

### 4. **国际化支持**
- 支持多语言配置
- 便于本地化适配
- 统一的翻译管理

### 5. **性能优化**
- 避免重复创建对象
- 减少内存占用
- 提高应用性能

这些常量文件共同构成了应用的基础配置层，为整个应用提供了统一的配置管理，是现代前端应用架构中不可或缺的一部分。
