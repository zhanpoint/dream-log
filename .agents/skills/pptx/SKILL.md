## 设计思路

**不要做无聊的幻灯片。** 白底加纯项目符号打动不了人。每张片子都可从下列方向里找灵感。

### 动手前

- **选定大胆且贴合内容的配色**：调色盘要让人感觉是为**这个主题**量身定的。若把你的颜色换进另一套完全不相关的演示里依然「说得通」，说明还不够具体。  
- **主次分明，不要平均**：一种颜色应占主导（约 60–70% 视觉权重），1～2 种辅助色，再加一个锐利点缀。**不要让所有颜色平分秋色**。  
- **明暗对比**：标题页与结尾用深色，正文用浅色（「三明治」结构）；或全程深色走高级质感路线。  
- **坚持一个视觉母题**：选定**一种**有辨识度的元素并贯穿——圆角图片框、彩色圆里的图标、单侧粗边框等，每张都沿用。  

### 配色方案

按主题选色，不要默认「通用蓝」。下表可作灵感（色值为十六进制）：

|------|------|------|------|
| **Midnight Executive** | `1E2761`（海军蓝） | `CADCFC`（冰蓝） | `FFFFFF`（白） |
| **Forest & Moss** | `2C5F2D`（森林绿） | `97BC62`（苔绿） | `F5F5F5`（奶油白） |
| **Coral Energy** | `F96167`（珊瑚） | `F9E795`（金） | `2F3C7E`（海军蓝） |

| **Ocean Gradient** | `065A82`（深蓝） | `1C7293`（青绿） | `21295C`（午夜蓝） |
| **Charcoal Minimal** | `36454F`（炭灰） | `F2F2F2`（米白） | `212121`（黑） |

### 每一张幻灯片

**每张都需要视觉元素**——图、表、图标或形状。纯文字片很容易被忘掉。

**版式可选：**
- 双栏（左文右图）  
- 图标 + 文字行（图标放在彩色圆里，粗标题，下方说明）  
- 2×2 或 2×3 网格（一侧大图，另一侧内容块网格）  
- 半幅出血图（整侧铺满）+ 叠在上面的内容区  

**数据展示：**
- 大号数字强调（60–72pt 大数字 + 下方小标签）  
- 对比列（前后、利弊、并排方案）  
- 时间线或流程（编号步骤、箭头）  

**视觉润色：**
- 小节标题旁放小彩色圆 + 图标  
- 关键数字或 slogan 用斜体点缀  

### 字体

**选一组有意思的字体搭配**——不要默认 Arial。标题字要有性格，正文用干净易读的字族。

| 标题字体 | 正文字体 |
|----------|----------|
| Georgia | Calibri |
| Arial Black | Arial |
| Calibri | Calibri Light |
| Cambria | Calibri |
| Trebuchet MS | Calibri |
| Impact | Arial |
| Palatino | Garamond |
| Consolas | Calibri |

| 元素 | 字号 |
|------|------|
| 幻灯片标题 | 36–44pt 粗体 |
| 小节标题 | 20–24pt 粗体 |
| 正文 | 14–16pt |
| 说明/脚注 | 10–12pt，略弱 |

### 间距

- 页边距至少 **0.5 英寸**  
- 内容块之间 **0.3–0.5 英寸**  
- 留出呼吸感——不要铺满每一寸  

### 避免（常见错误）

- **不要每张版式都一样**——栏宽、卡片、强调框要穿插变化  
- **不要把正文居中**——段落与列表左对齐；仅标题可居中  
- **不要字号对比不足**——标题至少 36pt+，才能和 14–16pt 正文拉开层次  
- **不要默认蓝色**——按具体主题选色  
- **不要随意混用间距**——选定 0.3" 或 0.5" 等规则并一贯执行  
- **不要只美化一张、其余敷衍**——要么整套做到位，要么全程保持简单一致  
- **不要做纯文字片**——加图、图标、图表或形状；避免只有标题 + 项目符号  
- **不要忘记文本框内边距**——若要让线条与文字边缘对齐，给文本框设 `margin: 0`，或把形状外移以抵消内边距  
- **不要用低对比元素**——图标与文字相对背景都要有足够对比；避免浅底浅字、深底深字  
- **绝不要在标题下加装饰线**——这是典型「AI 幻灯片」痕迹；用留白或背景色分区即可

## 质检（必做）

**默认认为有问题，你的任务是把它找出来。**

第一版渲染几乎不会一次完美。把质检当成**找 bug**，不是走过场。若第一眼「零问题」，多半是你看得不够细。

### 内容质检

```bash
python -m markitdown output.pptx
```

检查缺内容、错别字、顺序错误。

**使用模板时，检查是否残留占位符：**

```bash
python -m markitdown output.pptx | grep -iE "xxxx|lorem|ipsum|this.*(page|slide).*layout"
```

若 grep 有输出，修完再宣告成功。

### 视觉质检

**⚠️ 使用子代理（Subagents）**——即便只有 2～3 张。你已盯着代码太久，容易「看见预期而非真实」。子代理是新鲜视角。

先将幻灯片转为图片（见下文 [转为图片](#转为图片)），再使用类似提示：

```
请逐张目视检查这些幻灯片。默认认为存在问题，请找出来。

关注：
- 元素重叠（文字压到形状上、线穿过字、层叠错乱）
- 文字溢出或在边缘/文本框处被裁切
- 装饰线按单行标题对齐，但标题实际折成两行
- 来源引用或页脚与上方内容相撞
- 间距过近（小于约 0.3"）或卡片/区块几乎贴在一起
- 疏密不均（一处大片空白、另一处挤成一团）
- 距幻灯片边缘边距不足（小于约 0.5"）
- 多列或同类元素纵向/横向未对齐
- 文字对比度不足（如浅灰字配奶油色底）
- 图标对比度不足（如深色图标直接放在深色底上且无反差色衬底）
- 文本框过窄导致过度折行
- 残留占位内容

每张幻灯片请列出问题或可改进点，即便较轻微。

请阅读并分析下列图片：
1. /path/to/slide-01.jpg（预期：[简短说明]）
2. /path/to/slide-02.jpg（预期：[简短说明]）

报告所有发现的问题，包括轻微问题。
```

### 验证闭环

1. 生成幻灯片 → 转图片 → 检查  
2. **列出发现的问题**（若写「无」，请更苛刻地再看一遍）  
3. 修复  
4. **对受影响的幻灯片再次验证**——修一处常会引出另一处  
5. 重复，直到完整过了一轮仍无新问题  

**在完成至少一轮「修复 → 再验证」之前，不要宣称成功。**

## 转为图片

将演示转为逐页图片以便目视检查：

```bash
python scripts/office/soffice.py --headless --convert-to pdf output.pptx
pdftoppm -jpeg -r 150 output.pdf slide
```

会生成 `slide-01.jpg`、`slide-02.jpg` 等。

修复后若只需重渲特定页：

```bash
pdftoppm -jpeg -r 150 -f N -l N output.pdf slide-fixed
```

## 依赖

- `pip install "markitdown[pptx]"` — 文本提取  
- `pip install Pillow` — 缩略图网格

- LibreOffice（`soffice`）— 转 PDF（沙箱环境可通过 `scripts/office/soffice.py` 自动配置）
