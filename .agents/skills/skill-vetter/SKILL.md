# Skill Vetter

面向 AI 代理技能、安全优先的审查协议。**未经审查，切勿安装技能。**

## 何时使用

- 在安装并运行来自任意技能之前
- 在评估其他代理分享的技能时
- 在任何时候被要求安装未知代码时

## 审查协议

### 第一步：来源核查

```
需回答的问题：
- [ ] 该技能来自何处？
- [ ] 作者是否知名/可信？
- [ ] 下载量/星标有多少？
- [ ] 最近一次更新是何时？
- [ ] 是否有其他代理的评价？
```

### 第二步：代码审查（强制）

阅读技能内的**全部**文件。对照以下**危险信号**检查：

```
若出现下列情况，应立即拒绝：
─────────────────────────────────────────
• 对未知 URL 使用 curl/wget
• 向外部服务器发送数据
• 索要凭据/令牌/API 密钥
• 无正当理由读取 ~/.ssh、~/.aws、~/.config
• 访问 MEMORY.md、USER.md、SOUL.md、IDENTITY.md
• 对任意内容使用 base64 解码
• 对外部输入使用 eval() 或 exec()
• 修改工作区外的系统文件
• 安装软件包却未在清单中列出
• 向 IP 而非域名发起网络调用
• 混淆代码（压缩、编码、最小化）
• 请求提升权限/sudo
• 访问浏览器 Cookie/会话
• 触碰凭据类文件
─────────────────────────────────────────
```

### 第三步：权限范围

```
评估项：
- [ ] 需要读取哪些文件？
- [ ] 需要写入哪些文件？
- [ ] 会执行哪些命令？
- [ ] 是否需要网络访问？访问何处？
- [ ] 相对其宣称目标，范围是否最小化？
```

### 第四步：风险分级

| 风险等级 | 示例 | 处理 |
|----------|------|------|
| 🟢 低 | 笔记、天气、格式化 | 基础审查后可安装 |
| 🟡 中 | 文件操作、浏览器、API | 须完整代码审查 |
| 🔴 高 | 凭据、交易、系统级 | 须人工批准 |
| ⛔ 极高 | 安全配置、root 访问 | **不要安装** |

## 输出格式

审查结束后，按下列格式产出报告：

```
技能审查报告
═══════════════════════════════════════
技能：[名称]
来源：[ClawdHub / GitHub / 其他]
作者：[用户名]
版本：[版本]
───────────────────────────────────────
指标：
• 下载量/星标：[数量]
• 最近更新：[日期]
• 已审文件数：[数量]
───────────────────────────────────────
危险信号：[无 / 列出]

所需权限：
• 文件：[列表或「无」]
• 网络：[列表或「无」]
• 命令：[列表或「无」]
───────────────────────────────────────
风险等级：[🟢 低 / 🟡 中 / 🔴 高 / ⛔ 极高]

结论：[✅ 可安全安装 / ⚠️ 谨慎安装 / ❌ 切勿安装]

备注：[任何说明]
═══════════════════════════════════════
```

## 快速审查命令

针对托管在 GitHub 上的技能：

```bash
# 查看仓库统计
curl -s "https://api.github.com/repos/OWNER/REPO" | jq '{stars: .stargazers_count, forks: .forks_count, updated: .updated_at}'

# 列出技能目录下文件
curl -s "https://api.github.com/repos/OWNER/REPO/contents/skills/SKILL_NAME" | jq '.[].name'

# 拉取并审阅 SKILL.md
curl -s "https://raw.githubusercontent.com/OWNER/REPO/main/skills/SKILL_NAME/SKILL.md"
```

## 信任层级

1. **高星仓库（1000+）** → 中等审查
2. **已知作者** → 中等审查
3. **新来源/未知来源** → 最严审查
4. **索要凭据的技能** → 始终须人工批准

## 请牢记

- 没有任何技能值得以牺牲安全为代价
- 存疑则不安装
- 高风险决策请交给你的人类用户
- 将审查结果记录下来，供日后查阅
